Spring Boot整合Spring Security

发表于 2020-07-17 更新于 2020-09-25 分类于 框架 ， Spring Boot 阅读次数： Valine：
本文字数： 6.9k 阅读时长 ≈ 6 分钟

本文记录了Spring Boot整合Spring Security以及Spring Security的相关配置

版本

Spring Security 5.3.3

Spring Boot 2.3.1

创建项目

在idea中新建项目时勾选Spring Security

当不作任何配置的时候，Spring Boot会自动配置Spring Security

启动项目后会在控制台输出一个临时密码以供登录，这是默认的配置

如果需要详细配置，需要新建一个继承自WebSecurityConfigurerAdapter的SpringSecurityConfig配置类

通过重写configure方法来配置Spring Security,该方法有几种不同的重载

  • AuthenticationManagerBuilder参数，用于配置权限角色
  • HttpSecurity参数，用于配置http登录拦截

示例用到的表和类

本文展示的Demo主要用到3个表，

  • user表，存放用户信息，映射类User
    • 字段：id, username, password, enabled…
  • role表，存放角色信息，映射类Role
    • 字段：id, name(“ROLE_xxx”), description
  • user_role表，存放用户权限信息，一个用户可以有多个权限，映射类UserRole
    • 字段：id，uid，rid

配置权限角色

一般有3种配置方式：

  1. 在properties中配置，比较简单且使用较少这里省略
  2. 在内存中配置
  3. 在数据库配置

A.在内存中配置

重写configure(AuthenticationManagerBuilder auth)方法

在Spring Security 5.3.3版本中，角色密码必须加密，我们可以使用Spring Security提供的BCryptPasswordEncoder，将其注入使用

B.在数据库配置

重写configure(AuthenticationManagerBuilder auth)方法

在数据库中配置需要先完成以下几步，

  1. 数据库映射的用户类继承UserDetails接口并重写方法
  2. 新建一个Service类继承UserDetailsService接口并重写方法
  3. 在configure方法中添加auth.userDetailsService(userDetailsService);
